# KODA.md — Инструкции для работы с проектом

## Обзор проекта

**Название:** nextjs_tailwind_shadcn_ts (версия 0.2.0)

**Тип проекта:** Веб-приложение на Next.js для сравнения кредитных предложений (финансовый маркетплейс)

Это полнофункциональное веб-приложение для управления и отображения кредитных предложений от различных МФО (микрофинансовых организаций). Проект включает административную панель для управления предложениями, публичный интерфейс для посетителей, систему синхронизации с внешними API и SEO-оптимизацию.

## Сборка и запуск

### Предварительные требования

Для работы проекта необходимы:
- **Bun** (рекомендуется) или **Node.js** 18+
- **npm** или **yarn** (если не используется Bun)

### Установка зависимостей

`ash
# Рекомендуется использовать Bun
bun install

# Альтернатива через npm
npm install
`

### Запуск в режиме разработки

`ash
# С использованием Bun
bun dev

# С использованием npm
npm run dev
`

После запуска приложение будет доступно по адресу: **http://localhost:3000**

### Сборка для продакшена

`ash
# С использованием Bun
bun run build

# С использованием npm
npm run build
`

### Запуск продакшен-сервера

`ash
# С использованием Bun
bun run start

# С использованием npm
npm run start
`

## Конфигурация веб-сервера (Caddy)

Проект использует **Caddy** в качестве обратного прокси-сервера. Конфигурация находится в файле Caddyfile.

- **Порт 81** — основной порт для веб-приложения
- Автоматическое проксирование запросов на **localhost:3000** (Next.js)
- Поддержка параметра XTransformPort для динамического проксирования на другие порты

### Запуск Caddy

`ash
caddy run
`

## База данных

### Используемая СУБД

**PostgreSQL** — объектно-реляционная система управления базами данных. Подключение через переменную окружения DATABASE_URL.

### Схема данных

Основные модели в базе данных:

#### Аутентификация (Auth.js)
1. **User** — пользователи системы (роли: admin, editor, user)
2. **Account** — аккаунты OAuth провайдеров
3. **Session** — сессии пользователей
4. **VerificationToken** — токены верификации

#### Основной бизнес
5. **LoanOffer** — кредитные предложения от МФО
6. **Tag** — теги для категоризации предложений
7. **OfferTag** — связь многие-ко-многим между предложениями и тегами

#### Личный кабинет пользователя
8. **FavoriteOffer** — избранные предложения пользователей
9. **SearchHistory** — история поиска пользователей
10. **GuestMigration** — лог миграции данных гостей

#### Блог
11. **BlogCategory** — категории блога
12. **BlogAuthor** — авторы статей
13. **BlogPost** — статьи блога
14. **BlogTag** — теги статей
15. **BlogPostTag** — связь статьи с тегами

#### Programmatic SEO
16. **SeoCity** — города для SEO-страниц
17. **SeoLoanType** — типы займов для SEO
18. **SeoPage** — программные SEO-страницы
19. **SeoFaq** — FAQ-блоки для SEO-страниц

#### Системные модели
20. **AuditLog** — журнал аудита изменений
21. **SyncLog** — журнал синхронизации с внешними API
22. **SeoTemplate** — SEO-шаблоны для генерации мета-тегов
23. **Setting** — настройки сайта

### Переменные окружения

Для подключения к базе данных используется переменная DATABASE_URL.

## Стиль кодирования

### TypeScript

Проект использует строгую типизацию с некоторыми послаблениями:
- noImplicitAny: false — допускается использование any
- ignoreBuildErrors: true в Next.js — игнорирование ошибок типов при сборке

### Импорты

В проекте настроены алиасы для импортов:
- @/* — ./src/*
- @/components — ./src/components
- @/lib — ./src/lib
- @/hooks — ./src/hooks
- @/components/ui — ./src/components/ui

### UI-компоненты

Проект использует библиотеку shadcn/ui со стилем new-york:
- Компоненты располагаются в src/components/ui/

## Особенности проекта

### Аутентификация

Используется NextAuth для аутентификации пользователей. Система ролей:
- **admin** — полный доступ к управлению
- **editor** — может редактировать предложения
- **user** — обычный пользователь

### Синхронизация с внешними API

Система поддерживает синхронизацию кредитных предложений из внешних источников (Leads.su, Admitad и др.). Каждая синхронизация логируется в модели SyncLog.

### SEO

Проект имеет встроенную систему SEO-оптимизации:
- Мета-теги для каждого предложения
- Шаблоны для генерации описаний (SeoTemplate)
- Поддержка кастомных описаний для каждого предложения

### Аудит

Все изменения в предложениях логируются в модели AuditLog, включая:
- Автора изменения
- Дату и время
- Изменённые поля
- Старое и новое значения
